# Risks and Mitigation in Vibe-Research

Understanding the dangers and how to address them.

## Core Risks

### 1. AI Hallucinations

**问题**: AI 可能生成虚假信息、不存在的引用、错误数据

**表现**:
- 引用不存在的论文
- 编造实验数据
- 错误陈述研究发现

**缓解措施**:
```
✅ 所有声明必须有可追溯来源
✅ 使用 RAG（检索增强生成）
✅ 人工验证关键引用
✅ 交叉检查 AI 输出与原始文献
```

---

### 2. Bias Amplification

**问题**: AI 训练数据中的偏见会被放大

**表现**:
- 倾向于主流观点，忽略边缘但有价值的研究
- 地域/语言偏见（英语论文优先）
- 领域偏见（某些领域研究过多）

**缓解措施**:
```
✅ 故意包含多样化的来源
✅ 人工审查假设的多样性
✅ 明确记录潜在的偏见来源
✅ 鼓励探索非主流方向
```

---

### 3. Reproducibility Crisis

**问题**: AI 生成的研究可能难以复现

**表现**:
- 代码有 bug 或不完整
- 实验步骤记录不清
- 随机种子未固定

**缓解措施**:
```
✅ 完整记录所有步骤（AI 日志）
✅ 代码版本控制
✅ 预注册实验协议
✅ 人工审核方法部分
```

---

### 4. Skill Erosion

**问题**: 研究者可能失去核心技能

**表现**:
- 不再能独立设计实验
- 丧失批判性阅读能力
- 过度依赖 AI 判断

**缓解措施**:
```
✅ 保持"手动手"练习
✅ 将 AI 输出作为起点，不是终点
✅ 培养验证 AI 的习惯
✅ 继续学习基础方法论
```

---

### 5. Information Overload (Irony)

**问题**: AI 加速研究 → 论文更多 → 更难跟上

**表现**:
- AI 生成大量低质量论文
- 同行评审负担加重
- 真正有价值的发现被淹没

**缓解措施**:
```
✅ AI 辅助筛选和总结
✅ 提高发表门槛（质量 > 数量）
✅ 建立信任标记系统
✅ 专注于关键验证，不是数量
```

## Multi-Layer Mitigation Framework

### Technical Layer

```
┌─────────────────────────────────────┐
│         技术保障                      │
├─────────────────────────────────────┤
│ • RAG 确保来源可追溯                  │
│ • 知识图谱验证逻辑一致性               │
│ • 自动化事实核查工具                   │
│ • 版本控制和实验日志                   │
└─────────────────────────────────────┘
```

### Policy Layer

```
┌─────────────────────────────────────┐
│         政策保障                      │
├─────────────────────────────────────┤
│ • 透明披露 AI 参与程度                 │
│ • 人工审核关键步骤                    │
│ • 发表前复现性检查                    │
│ • AI 辅助研究的伦理准则               │
└─────────────────────────────────────┘
```

### Human Layer

```
┌─────────────────────────────────────┐
│         人工保障                      │
├─────────────────────────────────────┤
│ • 假设选择：人做最终决定               │
│ • 方法审核：确保实验合理               │
│ • 结果解释：提供领域洞察               │
│ • 最终把关：对输出负责                │
└─────────────────────────────────────┘
```

---

## Human-in-the-Loop Checkpoints

```
Step 1: 知识缺口 → 人选择方向
Step 2: 文献综述 → 人验证覆盖度
Step 3: 假设生成 → 人评估价值★★★ 关键
Step 4: 实验设计 → 人审核可行性★★★ 关键
Step 5: 执行 → 人监督异常
Step 6: 分析 → 人验证统计方法★★★ 关键
Step 7: 写作 → 人最终审核★★★ 关键
```

## The Swiss Cheese Model for Vibe-Research

```
        AI生成内容
             │
    ┌────────▼────────┐
    │  自动化事实检查  │← 部分问题被拦截
    └────────┬────────┘
             │
    ┌────────▼────────┐
    │  引用验证工具   │← 部分问题被拦截
    └────────┬────────┘
             │
    ┌────────▼────────┐
    │  同行评审流程   │← 部分问题被拦截
    └────────┬────────┘
             │
    ┌────────▼────────┐
    │  复现性检查     │← 部分问题被拦截
    └────────┬────────┘
             │
    ┌────────▼────────┐
    │  社区监督      │← 最终保障
    └─────────────────┘

没有单一层能捕获所有问题，多层组合更可靠。
```

---

## Key Principle

> **Vibe-research 的最佳结果来自 AI-人协作，不是 AI 单独工作。**

人的角色从"执行者"变成"把关者"—— 这不是降级，而是升级。
