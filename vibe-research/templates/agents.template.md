## Workflow Priority

1. Understand the question and current state before editing.
2. Prefer project files over chat memory.
3. Work from explicit tasks with named inputs and deliverables.
4. Keep evidence, analysis, and prose aligned.

## Fail-Safety

- Do not fabricate citations or empirical results.
- Do not overwrite raw data.
- Distinguish evidence summary from interpretation.
- Distinguish descriptive findings from causal claims.
- If context is missing, state assumptions explicitly.

## Writing Rules

- Preserve citation keys and reference integrity.
- Avoid overstating novelty or contribution.
- Keep claims matched to code, tables, figures, or retrieved sources.

## Logging Rules

- Update `STATE.md` after meaningful work sessions.
- Update `TASKS.md` when task status changes.
- Append major edits or decisions to `CHANGELOG.md`.
- Use Git for diff history and rollback points, not as the only source of current project state.

## Coordination

- Codex and Claude Code should read the same project files.
- Do not create separate memory systems for different agents.
