# Phase 1: PubMed Retrieval (Detailed Guide)

## Overview

Phase 1 retrieves literature from PubMed using batch processing with curl and proper rate limiting.

## Execution

### Basic Usage

```bash
# Run retrieval with search mapping
python scripts/pubmed_batch_retrieval.py process/search_mapping.json

# Resume from checkpoint if interrupted
python scripts/pubmed_batch_retrieval.py process/search_mapping.json --resume

# With API key for higher rate limit (10 req/sec)
python scripts/pubmed_batch_retrieval.py process/search_mapping.json --api-key YOUR_KEY

# Specify output file
python scripts/pubmed_batch_retrieval.py process/search_mapping.json --output process/phase1_results.json
```

## Input: search_mapping.json

Generated by Phase 0 or created manually:

```json
{
  "section": "1.1",
  "outline_title": "Smoking and Cognitive Function",
  "date_range": "2022:2025[Date - Publication]",
  "themes": [
    {
      "theme": "Smoking_Cognitive_Performance",
      "maps_to": ["1.1.1"],
      "query": "(smoking[Title/Abstract] OR tobacco[Title/Abstract]) AND (cognit*[Title/Abstract])",
      "retmax": 100
    }
  ]
}
```

Enables resume capability by tracking completed themes.

## Key Features

### 1. Batch Processing

- Processes 200 PMIDs per EFetch request
- Respects PubMed rate limits (3 req/sec without API key, 10 req/sec with)
- Uses `curl` for reliable API calls

### 2. Complete Data Extraction

- **Abstracts**: Uses `findall()` instead of `findtext()` for 100% coverage
- **DOIs**: Explicit extraction from ArticleId elements
- **Full metadata**: Volume, issue, pages, publication types, MeSH terms

### 3. Resume Capability

If retrieval is interrupted:
```bash
# Re-run with --resume flag
python scripts/pubmed_batch_retrieval.py process/search_mapping.json --resume
```

Skips already-processed themes and continues from checkpoint.

## Rate Limiting

| Mode | Rate Limit | Delay |
|:-----|:-----------|:------|
| Without API key | 3 requests/second | 0.34s |
| With API key | 10 requests/second | 0.10s |

## Quality Checklist

After retrieval, verify:
- [ ] All themes have results
- [ ] Average abstract length > 500 chars
- [ ] No duplicate PMIDs
- [ ] DOIs extracted (90%+ coverage)
- [ ] JSON format valid

## Troubleshooting

### Empty abstracts

**Symptom**: Many articles missing abstract data

**Solution**: Script uses `findall()` for complete extraction. If issue persists, check if query includes `hasabstract[text]`.

### Missing DOIs

**Symptom**: < 50% DOIs extracted

**Solution**: Some articles don't have DOIs. Expect 70-90% coverage depending on field.

### Rate limit errors

**Symptom**: "429 Too Many Requests" errors

**Solution**:
1. Use `--api-key` for higher limits
2. Script includes 0.34s delay between requests
3. If errors persist, increase delay in script
